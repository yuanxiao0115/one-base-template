# Layer Placement

## 迁移落点映射

| 老项目关注点                 | 目标落点                                       | 规则                           |
| ---------------------------- | ---------------------------------------------- | ------------------------------ |
| 后端路径差异、字段映射       | `packages/adapters` 或模块内 `api.ts`/`compat` | 不把原始后端字段散落进页面模板 |
| 鉴权流程、菜单权限、通用状态 | `packages/core`                                | 只放逻辑与契约，不引入具体 UI  |
| 布局壳、共享交互容器         | `packages/ui`                                  | 只做通用壳，不反向依赖 app     |
| 页面编排、模块路由、样式落地 | `apps/admin`                                   | 只做页面组装与模块装配         |
| 规则说明与长期文档           | `apps/docs/docs/guide/*`                       | 与代码同变更维护               |

## 路由 / 菜单 / 鉴权约束

- 路由保持前端静态声明，落在 `modules/**/routes.ts`。
- 不把老项目运行时 `addRoute` 依赖原样搬过来。
- 详情页、编辑页优先用 `meta.activePath`，不要滥用 `meta.skipMenuAuth`。
- `/sso` 回调、菜单树权限、首页放行策略必须服从当前模板规则，不回退历史分流启动。

## 页面编排约束

- 页面文件只保留编排；字段映射下沉到 `form.ts` 或语义明确的模块内文件。
- 老项目的“万能 helper / normalize / wrapper”只有在跨 3 处以上稳定复用时才允许保留。
- 命名优先短、清楚、通用，避免把旧业务名词包装成公共能力。

## 何时进入共享包

只有在满足以下条件时才下沉到共享包：

- 同类逻辑至少出现多处重复
- 语义已经稳定
- 下沉能显著降低维护成本

否则保留在目标模块附近实现。

## 常见禁止项

- 不把 `packages/core` 写成 UI 逻辑容器
- 不在 `apps/admin` 页面模板里做字段归一化
- 不把老项目的布局、主题、启动链路私货强行塞进模板
- 不因为迁移名义扩大到 `apps/portal/**` 或全局运行时配置
