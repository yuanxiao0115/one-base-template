## admin 未登录放行首页修复

- 根因：
  - `packages/core/src/stores/auth.ts` 在 `token` 模式下过度信任 `ob_auth_user` 缓存，只要本地存在用户缓存就直接判定已登录，没有校验 `tokenKey` 对应的本地 token。
- 修复：
  - `packages/core/src/createCore.ts` 新增 `auth.mode/tokenKey` 配置。
  - `packages/core/src/stores/auth.ts` 在初始化和 `ensureAuthed()` 中统一校验 token；缺 token 时清空缓存用户并返回未登录。
  - `apps/admin/src/bootstrap/{core.ts,index.ts}`、`apps/portal/src/bootstrap/index.ts`、`apps/template/src/bootstrap/index.ts` 统一向 core 传入 `authMode/tokenKey`。
  - 新增 `packages/core/src/stores/auth.test.ts` 覆盖“无 token + 残留缓存用户”回归场景。
- 同轮一并修正的历史门禁：
  - `packages/core/src/hooks/user-management-unique.test.ts` 模块路径失效。
  - `apps/admin/tests/router/assemble-routes.unit.test.ts` 固定路由断言口径落后于当前路由装配实现。
  - `packages/utils/src/storage/__tests__/local.test.ts` 假设 `window.localStorage.clear()` 在当前测试环境可直接调用。
  - `apps/docs/docs/.vitepress/config.ts` 的 readonly 数组类型与 VitePress 配置类型不兼容。
  - `apps/admin/src/services/auth/auth-avatar-preference-service.ts` 函数名不符合命名白名单。
