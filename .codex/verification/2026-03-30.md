## template：登录页 Element Plus 解析链路修复（追加）

### 范围

- `apps/template/build/index.ts`
- `apps/template/build/vite-plugins.ts`
- `apps/template/vite.config.ts`
- `apps/template/package.json`
- `apps/template/tests/architecture/vite-plugin-parity-source.unit.test.ts`
- `apps/template/AGENTS.md`
- `apps/docs/docs/guide/template-static-app.md`

### 结论

- 根因不是 `packages/ui/src/components/auth/LoginBox.vue` 自身表单逻辑，而是 template 的 Vite 仍停留在仅 `vue()` 的最小插件链，缺少和 admin 一致的 Element Plus 自动导入与组件解析。
- 在该链路缺失时，`LoginBox.vue` 里的 `el-form` 会退化为原生自定义标签，`ref` 拿到的不是 `FormInstance`，因此运行时触发 `formRef.value.validate is not a function`。
- 补齐 `createTemplatePlugins()` 后，template 已具备与 admin 一致的 Element Plus 解析入口，并新增源码门禁防止再次回退。

### 验证命令
