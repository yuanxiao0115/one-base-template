## tokenKey 污染修复（多子项目同域隔离）

### 范围

- `packages/core/src/config/platform-config.ts`
- `packages/core/src/config/platform-config.test.ts`
- `apps/admin/src/config/platform-config.ts`
- `apps/admin-lite/src/config/platform-config.ts`
- `apps/zfw-system-sfss/src/config/platform-config.ts`
- `apps/portal/public/platform-config.json`
- `apps/admin/tests/config/platform-config.unit.test.ts`
- `apps/admin-lite/tests/config/platform-config.unit.test.ts`
- `apps/zfw-system-sfss/tests/config/platform-config.unit.test.ts`
- `apps/docs/docs/guide/env.md`
- `README.md`

### 结论

- preset 模式下，`tokenKey/idTokenKey` 支持按 `storageNamespace`（未配置时回退 `appcode`）自动生成，默认形态为 `${scope}-token` 与 `${scope}-id-token`。
- `apps/admin`、`apps/admin-lite`、`apps/zfw-system-sfss` 已移除硬编码 `token/idToken`，统一走自动生成，避免多个子项目本地并行启动时 token 污染。
- `apps/portal` 运行时配置补充了 `storageNamespace=one-base-template-portal`，使 portal 的 token key 与其他子项目隔离。
- 三个应用的 `platform-config` 单测新增 key 断言，防止回归为通用 `token`。
