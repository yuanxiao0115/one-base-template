## puretable fork 第二批（tableKey / adaptive / pagination）

### 范围

- `packages/ui/src/components/table/internal/table-helpers.ts`
- `packages/ui/src/table-source.test.ts`
- `apps/docs/docs/guide/table-vxe-migration.md`

### 结论

- `ObTable` 已完成 `tableKey` registry 语义收口：默认 key 改为组件实例 `uid` 派生值，`props.tableKey` 可覆盖，`getTableRef()` 按 key 获取实例，避免同页多表默认 key 冲突。
- `adaptive` 已按 pure 预期切到“视口贴底优先 + 容器高度兜底”，并补齐 `ResizeObserver + window.resize` 监听与清理。
- `getTableDoms()` / `setHeaderSticky()` 已增强容错：优先 `$el` 根节点 + 多 selector fallback + fixed header 批量 sticky。
- 分页已区分 controlled 与 default 语义：受控 `currentPage/pageSize` 与 `defaultCurrentPage/defaultPageSize` 不再混用；`total > 0` 时不再透传 `pageCount`。
- 文档与源码门禁测试已同步更新。

## multi-role 走查后补强（expandSlot / formatter / lazy-tree）

### 范围

- `packages/ui/src/components/table/Table.vue`
- `packages/ui/src/components/table/internal/table-helpers.ts`
- `packages/ui/src/table-source.test.ts`
- `apps/docs/docs/guide/table-vxe-migration.md`

### 结论

- `expandSlot` 语义已补强：`type='expand'` 支持 `expandSlot` 与 `slots.expand` 双链路，避免误落普通字段渲染。
- `formatter` 语义已补齐：支持 pure 4 参签名与 params 签名，修复类型契约有定义但运行时未生效的问题。
- lazy tree `load` 返回值已统一经过 `normalizeTreeRows`，修复首层与子层归一化规则不一致。
- 默认单元格渲染已覆盖 boolean/bigint 文本化，避免布尔值显示为空白。
- 风险清单已写入文档，含 DOM 结构耦合、性能热点和列拖拽能力边界。

### 验证结果

- `packages/ui`：源码测试 `12/12` 通过；`typecheck/lint/build` 通过。
- `apps/admin`：6 个源码测试文件 `8/8` 通过；`typecheck` 通过。
- `apps/docs`：`lint` 0 warning / 0 error；`build` 成功。

### 仍需后续优化（非阻断）

- tooltip 批量实例在大表格场景的渲染开销（性能走查 P1）。
- adaptive 在高频 resize 场景下的观察器重绑与 `doLayout` 热点（性能走查 P1）。
- 统一壳组件暂未形成“列拖拽排序”标准能力（`ObTable` 不支持，`ObVxeTable` 仅透传扩展入口）。
