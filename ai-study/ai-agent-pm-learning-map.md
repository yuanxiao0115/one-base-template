# AI Agent 产品经理学习地图（每天 3 小时）

## TL;DR

- 学习周期：**12 周（2026-03-11 至 2026-06-02）**。
- 时间投入：**每天 3 小时**，总计约 **252 小时**。
- 学习目标：从“会用 AI 工具”升级为“可独立定义、评估并推进 Agent 产品落地”的 AI Agent PM。

## 背景与目标

你当前已具备较强工程化能力与 AI Coding 使用经验，短板不在“写代码”，而在“Agent 产品方法论 + 指标化运营 + 跨角色推进”。

本学习地图目标是让你在 12 周内具备以下能力：

- 能选择高 ROI 的 Agent 场景并定义边界。
- 能搭建从需求、评测、上线到复盘的闭环。
- 能用业务语言汇报价值（效率、质量、成本、风险）。

## 范围与非范围

### 范围

- Agent 产品设计与需求拆解。
- 工具调用、RAG、工作流、记忆等产品侧常识。
- 评测体系、上线策略、安全合规、运营迭代。

### 非范围

- 模型预训练与深度学习底层推导。
- 大规模分布式训练与底层推理框架优化。
- 纯学术论文导向路径。

## 前置条件

- 基础环境：MacBook Pro M1 16G（可用）。
- 开发工具：`VS Code`、`Node.js`、`pnpm`、`Postman/Insomnia`、`Notion/飞书文档`。
- API 能力：至少一个可用的模型 API Key。
- 项目记录：建议建立 `学习日志 + 指标看板`（每周更新）。

## 每天 3 小时固定节奏（建议）

- `00:00-00:45` 输入学习：官方文档/案例拆解。
- `00:45-01:45` 动手实操：需求稿、流程图、PRD、评测样本或原型。
- `01:45-02:30` 评测复盘：记录问题、误答样本、风险点。
- `02:30-03:00` 输出沉淀：产出日报（结论 + 下步动作）。

## 每周 7 天学习模板（可循环）

- Day1：主题概念学习 + 拆 1 个真实案例。
- Day2：把概念转成你的业务场景卡片（至少 3 个）。
- Day3：写 1 份产品文档（PRD/流程图/边界定义）。
- Day4：做 1 轮评测设计（样本、指标、打分口径）。
- Day5：做 1 轮迭代方案（性能/成本/准确率/风险）。
- Day6：做 1 次模拟评审（产品/研发/测试视角）。
- Day7：周复盘与下周计划（保留、删除、新增事项）。

## 验证与验收

每周末自检以下 6 个指标：

- 任务成功率（Task Success Rate）
- 一次解决率（First Pass Resolution）
- 误答率（Hallucination/Error Rate）
- 平均耗时（Latency）
- 单次成本（Cost per Task）
- 使用率与留存（Adoption/Retention）

通过标准：

- 每周至少有 **1 个可复用文档产出**。
- 每两周至少有 **1 次指标改善**。
- 第 12 周形成 **可复用 Playbook** 并可讲清楚方法论。

## 回滚方案（学习计划执行失败时）

若连续 2 周未完成计划目标，执行以下回滚：

1. 砍掉低优先级主题，只保留“场景选择、评测、上线”三大主线。
2. 每周目标从 3 个降到 1 个，确保完成率。
3. 所有学习内容改为“围绕一个真实试点项目”推进，停止泛学。

## FAQ

### Q1：需要本地跑大模型吗？

不需要。当前阶段以“产品能力落地”为主，优先使用云端模型。可本地跑轻量模型用于联调。

### Q2：M1 16G 够吗？

够。做 Agent PM 和应用层验证完全够用。

### Q3：每天只有 2 小时怎么办？

把“输入学习”压缩到 30 分钟，保留“实操 + 复盘 + 输出”。优先保证产出不断。
