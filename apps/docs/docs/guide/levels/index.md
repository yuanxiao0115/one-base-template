# 按水平进入（P2 / P4 / P6）

## TL;DR

- 文档主入口固定为按能力水平分层：`P2（上手） -> P4（独立开发） -> P6（架构治理）`。
- 角色入口（使用者/维护者）只做辅助筛选，不作为第一入口。
- 先选层级再开始阅读，能最快拿到“可执行路径 + 验收命令”。

## 适用范围

- 适用于第一次进入仓库，不确定该从哪页开始读的同学。
- 适用于要快速做出“今天看哪条文档主线”的决策场景。
- 不适用于直接查单一 API 细节（请转对应专题页）。

## 前置条件

1. 你知道自己当前目标：首跑、独立交付，还是治理演进。
2. 你可以执行基础命令（至少 `pnpm` 可用）。
3. 你愿意先按分层路径读一页，再进入专题页。

## 30 秒入口决策

| 你的当前目标         | 先进入层级页                            | 完成后下一步                       |
| -------------------- | --------------------------------------- | ---------------------------------- |
| 第一次接手，先跑起来 | [P2 路线（上手）](/guide/levels/p2)     | 回到模块/路由专题页完成首个闭环    |
| 能独立开发，想少返工 | [P4 路线（独立开发）](/guide/levels/p4) | 按任务索引进入模块、路由、布局规范 |
| 负责规则与架构治理   | [P6 路线（架构治理）](/guide/levels/p6) | 进入 AGENTS、治理与发布文档主线    |

## 最短执行路径

1. 先从上表选一个层级入口，只读这一层，不跨层混读。
2. 在层级页按“最短执行路径”跑完一轮命令验证。
3. 遇到具体任务（如路由、模块、门户）再跳到对应专题页。
4. 若 10 分钟仍无法开工，说明层级选高了，退回上一层。

## 验证与验收

通过标准：

1. 你能明确说出自己当前处于 P2/P4/P6 哪一层。
2. 你能按所选层级页完成至少一组验证命令。
3. 你不再依赖角色页作为第一入口。

## FAQ

### 我是维护者，能不能直接跳 `for-maintainers`？

可以看，但建议先过一遍 [P6 路线（架构治理）](/guide/levels/p6) 建立主线，再用角色页补充任务入口。

### 我是业务开发，是不是永远只看 P2？

不是。完成 P2 的首轮闭环后，尽快升级到 P4，避免长期停留在“能跑但不稳”的状态。

### 角色入口什么时候使用？

当你已经确定层级后，用角色页快速定位“同层下的高频任务文档”。
