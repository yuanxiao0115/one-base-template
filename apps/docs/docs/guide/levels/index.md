# 按水平进入（P2 / P4 / P6）

## TL;DR

- 文档主入口改为按能力水平分层：`P2（上手） -> P4（独立开发） -> P6（架构治理）`。
- 角色入口（使用者/维护者）仍保留，但作为辅助筛选，不再作为主阅读路径。
- 选择你当前最接近的层级开始，先跑通闭环，再补深度原理。

## 选择建议

| 层级 | 你当前状态                 | 推荐入口                                | 目标                       |
| ---- | -------------------------- | --------------------------------------- | -------------------------- |
| P2   | 第一次接手项目，先要跑起来 | [P2 路线（上手）](/guide/levels/p2)     | 30-45 分钟完成首个开发闭环 |
| P4   | 能独立开发功能，想减少返工 | [P4 路线（独立开发）](/guide/levels/p4) | 1 个迭代内稳定交付模块     |
| P6   | 负责架构与治理，需要控风险 | [P6 路线（架构治理）](/guide/levels/p6) | 建立规则、门禁与演进策略   |

## 阅读规则

1. 先按层级读，再按任务跳转到专题页。
2. 每个层级页都给出“最小必读 + 验证命令 + 升级条件”。
3. 如果阅读 10 分钟仍无法开始动手，说明层级选高了，退回上一层。

## 辅助入口

- 角色视角：[框架使用者阅读入口](/guide/for-users)、[仓库维护者阅读入口](/guide/for-maintainers)
- 总览页：[文档总览](/guide/)
