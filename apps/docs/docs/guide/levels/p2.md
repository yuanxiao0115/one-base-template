# P2 路线（上手）

## TL;DR

- 目标不是“看懂所有架构”，而是先跑通一次完整开发闭环。
- 按 `快速开始 -> admin-lite 基座 -> 菜单路由规范` 的顺序读，30-45 分钟内可完成首轮实践。
- 完成最小验证（`typecheck + lint + build`）后，再进入 P4。

## 适用范围

- 你第一次接手仓库，对目录和规则不熟。
- 你当前最需要的是“今天就能改一处并验证通过”。
- 你不需要先理解全部治理规则和历史迁移细节。

## 前置条件

1. 本机已安装并可使用 `pnpm`。
2. 已完成仓库依赖安装（`pnpm install`）。
3. 你可以在本地启动 `admin-lite`。

## 30-45 分钟目标

1. 本地跑起 `admin-lite`。
2. 通过脚手架创建一个新后台 app。
3. 对新 app 跑通最小验证命令。

## 最短执行路径

### 1. 先跑通基础链路

按顺序阅读：

1. [快速开始](/guide/quick-start)
2. [admin-lite 后台基座](/guide/admin-lite-base-app)
3. [菜单与路由规范（Schema）](/guide/menu-route-spec)

### 2. 直接执行一次完整练习

```bash
pnpm install
pnpm dev:admin-lite
pnpm new:app my-admin --preset standard
pnpm -C apps/my-admin typecheck
pnpm -C apps/my-admin lint
pnpm -C apps/my-admin build
```

### 3. 写页面时先守住三条基线

1. 模块入口契约：`index.ts（moduleMeta） + routes.ts`。
2. 列表页基线：`ObPageContainer + ObTableBox + ObTable`。
3. 提交前至少执行：`typecheck + lint + build`。

## 验证与验收

通过标准：

1. 新 app 能启动并进入首页。
2. `typecheck`、`lint`、`build` 全部通过。
3. 你能说清“模块入口文件”和“路由规则文档”分别在哪。

建议补充（可选）：

```bash
pnpm -C apps/docs lint
pnpm -C apps/docs build
```

当你同步改过文档时，再补跑 docs 校验。

## FAQ

### 我必须先读完全部文档再开始吗？

不用。先按本页给的阅读顺序完成一次闭环，再按问题回查专题页。

### 什么时候看 `for-users` 页面？

当你想按“角色视角”快速补知识时看；但主路径依旧是分层路线。

### 跑命令报错该从哪查？

先看报错对应命令，再回到 [快速开始](/guide/quick-start) 的失败处理和环境检查章节。

## 升级到 P4 的信号

满足以下 2 项后，建议进入 [P4 路线（独立开发）](/guide/levels/p4)：

1. 你可以独立新增一个简单管理模块（list/search/create/edit），并通过 `typecheck/lint/build`。
2. 你知道改路由、改模块、改验证时分别应该看哪份文档，不再依赖他人口头指路。

如果暂时不满足，继续按本页“最短执行路径”补齐首轮闭环，不要跳级。
