# P4 路线（独立开发）

## 适合对象

- 已能独立完成基础页面开发。
- 目标是“稳定交付 + 少返工 + 易维护”。

## 1 个迭代目标

1. 独立完成模块新增或改造。
2. 不突破架构红线与门禁。
3. 文档与代码同步更新。

## 必读主线

1. [/guide/module-system](/guide/module-system)
2. [/guide/menu-route-spec](/guide/menu-route-spec)
3. [/guide/layout-menu](/guide/layout-menu)
4. [/guide/admin-lite-agent-redlines](/guide/admin-lite-agent-redlines)
5. [/guide/development](/guide/development)

## 任务索引（按你要做的事进入）

1. 新增模块：先看 `module-system`，再看 `admin-lite-base-app` 的模块基线。
2. 改路由/权限：先看 `menu-route-spec`，重点校验 `meta.access` 与 route-policy。
3. 改列表交互：先看 `layout-menu`，保持 `ObTable` 编排一致。
4. 提测与合并：按 `development` 的命令清单执行验证。

## 默认验证命令

```bash
pnpm -C apps/admin-lite typecheck
pnpm -C apps/admin-lite lint
pnpm -C apps/admin-lite lint:arch
pnpm -C apps/admin-lite test:run
pnpm -C apps/admin-lite build
pnpm -C apps/docs lint
pnpm -C apps/docs build
```

## 升级到 P6 的信号

- 你能识别“规则该放哪一层（根/子项目）”。
- 你能把一次改动沉淀成可复用的团队规范，而不是只修当前问题。
